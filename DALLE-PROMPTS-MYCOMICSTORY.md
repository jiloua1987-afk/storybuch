## TECHNISCHE SPECS

### DALL-E Settings:
- **Größe:** 1024x1024 (quadratisch)
- **Stil:** Natural (nicht HD/Vivid)
- **Qualität:** Standard

### Nach der Generierung:
1. **Resize:** 512x512 px für Web
2. **Format:** PNG mit Transparenz (optional)
3. **Optimierung:** TinyPNG oder ähnlich
4. **Dateinamen:** 
   - `category-liebe.png`
   - `category-familie.png`
   - `category-urlaub.png`
   - `category-geburtstag.png`
   - `category-biografie.png`
   - `category-freundschaft.png`
   - `category-momente.png`
   - `style-action.png`
   - `style-emotional.png`
   - `style-humor.png`

## INTEGRATION IN STEP1STORY

### Aktueller Code (mit SVG Icons):
```tsx
// src/components/icons/CategoryIcons.tsx
```

### Nach DALL-E Generation:
```tsx
// Ersetze SVG durch Image-Komponenten
import Image from "next/image";

export function HeartIcon() {
  return <Image src="/icons/category-liebe.png" alt="" width={48} height={48} />;
}

export function FamilyIcon() {
  return <Image src="/icons/category-familie.png" alt="" width={48} height={48} />;
}

// ... etc
```

### Oder: Behalte SVG + füge 3D Icons als Option hinzu
```tsx
<div className="relative">
  {use3DStyle ? (
    <Image src="/icons/category-liebe.png" alt="" width={48} height={48} />
  ) : (
    <HeartIcon className="w-12 h-12 text-brand-500" />
  )}
</div>
```

## TIPPS FÜR BESTE ERGEBNISSE

1. **Konsistenz ist key:**
   - Verwende immer dieselben Stil-Keywords
   - Gleiche Farbpalette für alle Icons
   - Ähnliche Komplexität

2. **Iteriere:**
   - Generiere mehrere Varianten
   - Wähle die konsistentesten aus
   - Passe Prompts an basierend auf Ergebnissen

3. **Teste im Context:**
   - Platziere Icons in deinem UI
   - Prüfe Lesbarkeit bei kleiner Größe
   - Teste auf verschiedenen Hintergründen

4. **Fallback:**
   - Behalte aktuelle SVG Icons als Fallback
   - A/B Test: 3D vs. SVG Icons
   - User Feedback einholen
